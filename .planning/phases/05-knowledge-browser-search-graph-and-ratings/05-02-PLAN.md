<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Search is implemented as frontend-only text filtering | high | Add backend `/knowledge/search` route and Qdrant repository search method. |
| Search tests depend on external OpenAI or Qdrant availability | high | Inject fake embedding provider and fake Qdrant repository in tests. |
| Search results lose evidence traceability | high | Return payload fields for UUID, entity type, vault path, source, chunk ID, and relationship UUIDs. |
| Qdrant API version drift breaks runtime search | medium | Implement `query_points` and keep a narrow fallback to `search` if the installed client exposes only the older method. |
</threat_model>
