<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Paywalled or authenticated content is downloaded | high | Only use explicit public PDF URLs from provider metadata; no cookies, login, redirects to auth, or scraping bypasses. |
| Oversized PDFs exhaust memory or disk | high | Enforce `PAPER_MAX_PDF_MB` before saving and content-stream safeguards before extraction. |
| Parser creates empty chunks and later evaluation treats them as evidence | high | If extracted text is empty, status is `metadata_only` or `failed`, and no chunks are emitted. |
| PDF text extraction follows malicious source instructions | medium | Parser returns text only; LLM use remains behind untrusted input wrappers in Plan 05. |
</threat_model>
