## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Prompt injection from future transcripts/papers | high | Add `backend/app/safety/input_boundaries.py` with constants marking external text as untrusted and tests asserting the warning string exists. |
| Secrets logged or exposed by health endpoints | high | Health endpoint returns service names/status placeholders only; settings object must not stringify secret values in responses. |
| Store drift caused by inconsistent IDs | high | Use Pydantic UUID fields for canonical identity across entity, relationship, and vector payload schemas. |
| SDK calls scattered through backend | medium | MongoDB and Qdrant direct clients live only in repository modules for Phase 1. |
</threat_model>
