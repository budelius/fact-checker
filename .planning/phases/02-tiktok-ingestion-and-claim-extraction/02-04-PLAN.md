<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Unsupported/private URLs trigger platform access | high | API validates public TikTok URL before job execution and returns recoverable failed state for unsupported URLs. |
| Uploaded videos are treated as third-party/provider uploads by accident | high | Upload route stores files locally under `vault/raw/videos/`; transcription/provider calls remain disabled unless explicit config enables them. |
| Oversized or unsupported video files exhaust disk | high | Upload route validates MIME/extension and size before committing the raw artifact path. |
| Progress disappears after failure | medium | Persist job stage/artifact/error state before returning terminal failure. |
| Duplicate canonical records on retry | high | Persistence service separates job UUIDs from canonical entity UUIDs and uses external aliases for video dedupe logic. |
| Secrets leak into vault/logs | high | Persistence writes source URLs and aliases only; provider keys/settings are never serialized. |
| Phase 2 overclaims truth | high | Persistence writes `evidence_status: pending` and no supported/contradicted/mixed truth labels. |
</threat_model>
