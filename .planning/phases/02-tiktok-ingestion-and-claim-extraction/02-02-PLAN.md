<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Unauthorized media download | high | `TIKTOK_MEDIA_DOWNLOAD_ENABLED` defaults to false and compliance is checked before any media subprocess runs. |
| Arbitrary URL or SSRF-like retrieval | high | Adapter accepts only TikTok hostnames and rejects unsupported URLs before subprocess execution. |
| Lost transcript timestamps | high | Transcript parser stores segment start/end seconds instead of returning only cleaned plain text. |
| External metadata/captions treated as trusted | medium | Adapter returns raw external text as data only; later extraction plans must wrap it before LLM calls. |
| Dependency failure blocks dev/tests | medium | Tests mock `yt-dlp`, OpenCV, and media files; fixture/pasted transcript path remains independent. |
</threat_model>
