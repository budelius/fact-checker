<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Prompt injection from transcript, caption, OCR, or metadata text | high | `build_claim_prompt()` must call `wrap_untrusted_text()` for transcript and visual/source-clue text before provider calls. |
| Invalid LLM JSON creates false claims | high | Parse provider output through Pydantic and return a failed extraction state on JSON/schema errors. |
| Phase 2 implies truth or evidence support | high | `ExtractedClaim.evidence_status` remains `pending`; triage copy/status cannot use supported/contradicted/true/false labels. |
| Source candidates are mistaken for discovered papers | medium | `ResearchBasisTriage` stores candidate type/value/source only and never downloads or evaluates papers. |
</threat_model>
