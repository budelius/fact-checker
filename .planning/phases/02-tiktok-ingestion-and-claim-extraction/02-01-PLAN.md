## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Transcript or screenshot artifacts cannot be resolved later | high | Add explicit schema and vault folder/path contracts for transcripts and screenshots before persistence writes. |
| Job UUID is mistaken for canonical entity UUID | medium | DTOs must name `job_uuid` separately from `video_uuid`, `transcript_uuid`, `screenshot_uuid`, and `claim_uuid`. |
| Phase 2 claims imply truth evaluation | high | Claim DTO must restrict `evidence_status` to `pending`; no supported/contradicted/mixed labels in Phase 2 schemas. |
| External text enters schemas as trusted instructions | medium | DTO field names should identify source/provenance and later plans must use the untrusted-input wrapper before LLM processing. |
</threat_model>
